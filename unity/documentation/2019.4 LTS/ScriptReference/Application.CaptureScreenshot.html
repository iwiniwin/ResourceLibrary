Application.CaptureScreenshot

切换到手册
Obsolete public static void CaptureScreenshot (string filename);
Obsolete public static void CaptureScreenshot (string filename, int superSize);

参数

filename截屏文件要保存到的路径名称。
superSize分辨率的增加倍数。

描述

在路径 filename 捕获截屏并将其作为 PNG 文件。

使用 ScreenCapture.CaptureScreenshot 而非已弃用的 Application.CaptureScreenshot。

如果该图像文件已存在，则会被覆盖。图像写入的位置 可以包含一个目录/文件夹列表。例如， 在 macOS 上，PNG 文件可以写入 /tmp/ScreenGrab.png。如果没有目录/文件夹 列表，图像将写入 Project 文件夹。CaptureScreenshot() 也可以 从 Editor 使用，例如在自定义 EditorWindow 中。默认情况下， 截屏图像也将写入 Project 文件夹。此外，必须选中 Game View ，以便 Editor 屏幕捕获正常工作。

当 superSize 参数大于 1 时，将生成更大分辨率的 截屏。例如，传递 4 将使截屏比通常情况大 4x4。 这适用于生成用于打印的截屏。

在移动平台上，文件名附加到持久数据路径。 请参阅 Application.persistentDataPath 以了解更多信息。 在 Android 上，此函数立即返回。生成的截屏将稍后提供。

注意：如上所述，可以从 Editor 制作截屏。在此 情况下，只能捕获 Game View。如果已选中 Scene View，则不会写入 任何截屏。切换到 Game View 将导致写入 最近捕获的图像。请参阅 EditorWindow，了解如何创建基于编辑器的 捕获窗口。如果 Game View 是最新的且多次单击 Editor 按钮， 则将写入第一个生成的截屏。 Application.CaptureScreenshot 是一个游戏特定的功能。
