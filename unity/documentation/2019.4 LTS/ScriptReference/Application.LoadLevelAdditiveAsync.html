Application.LoadLevelAdditiveAsync

切换到手册
Obsolete public static AsyncOperation LoadLevelAdditiveAsync (int index);
Obsolete public static AsyncOperation LoadLevelAdditiveAsync (string levelName);

描述

在后台以异步方式另外加载关卡。

与 LoadLevelAsync 不同，LoadLevelAdditiveAsync 不会销毁 当前关卡中的对象。将新关卡的对象添加到 当前场景。这可用于创建连续的虚拟世界； 在这些虚拟世界中遍历环境时会加载更多的内容。

Unity 将在后台加载线程中全面加载场景中的所有资源和对象。 这使您可以创建一个完全串流的世界，您可以在其中根据玩家位置不断加载和 卸载世界的不同部分，而且不会在游戏中出现任何中断。

生成的 AsyncOperation 中的 isDone 变量可用于查询 关卡加载是否已完成。LoadLevelAdditiveAsync 的结果也可用于在协程中生成。

在构建播放器时，Unity 会自动优化资源，以便 LoadLevelAdditiveAsync 以线性方式 从磁盘加载并避免搜索时间。请注意，Unity Editor 中的后台加载性能 远低于独立构建。在 Editor 中，您可能还会遇到 比播放器更多的加载中断情况。

请参阅 事件函数的执行顺序，了解有关加载关卡后调用序列的更多信息。

另请参阅：AsyncOperation、Application.backgroundLoadingPriority、Application.LoadLevelAdditive、Application.LoadLevel、Application.LoadLevelAsync。
