使用 Unity Editor 对 Unity Cloud Build 进行 Perforce 配置

Unity Cloud Build 支持存储在 Perforce 代码仓库中的项目。

启用 Cloud Build

1.在 Services 窗口中的 Cloud Build 旁边，单击 OFF 按钮以将其切换为 ON 状态：

2.在 CLOUD BUILD 窗口中，单击 Cloud Build 开关为项目启用 Cloud Build。

3.在 Build History 窗口中，单击 Set up Cloud Build。

设置源代码控制托管类型

在 SOURCE CONTROL 服务窗口中：

1.从 SOURCE CONTROL HOST TYPE 下拉菜单中，选择 Perforce。 2.在 SOURCE CONTROL HOSTING URL 字段中，输入 Perforce 代码仓库的 URL。有关 URL 格式的信息，请参阅以下的 URL 语法__部分。 3.单击 Next__ 按钮。

URL 语法

要连接到代码仓库，必须指定 Perforce 服务器的 URL。您可以指定 URL 使用以下协议：

  • HTTPS
  • SSL

以下是有效 URL 的示例：

  • https://127.0.0.1:1667
  • ssl:127.0.0.1:167

Grant access

在 GRANT ACCESS 窗口中，输入用户名和密码，然后单击 Next。

设置运行时目标平台

要为应用程序配置运行时平台，请执行以下操作：

1.在 CHOOSE PLATFORM 窗口中，从 PLATFORM 下拉菜单选择构建平台，然后单击 Next。 2.在 TARGET SETUP 窗口中，从 CLIENT WORKSPACE 下拉菜单选择用于构建项目的 Perforce 工作空间。 3.在 PROJECT SUBFOLDER 字段中，输入包含 Unity 项目的文件夹；特别是 Assets 和 ProjectSettings 文件夹。该文件夹的路径通常类似于 NewGameProject/Src/UnityProject/。

注意：如果您的 Unity 项目不在代码仓库的根目录中，则必须填写 PROJECT SUBFOLDER 字段。

1.从 UNITY VERSION 下拉菜单中，选择要用于构建项目的 Unity 版本。 2.（可选）如果不希望 Cloud Build 在您对项目进行更改时自动构建项目，请取消选中 AUTO-BUILD 复选框。有关更多信息，请参阅 Automated Build Generation。
