UDP 控制台参考

本节提供有关 UDP 控制台的信息架构的简要概述。

UDP 控制台包含一个导航栏，可让你在 UDP 控制台的以下部分之间进行导航：

  • 开始使用
  • 我的游戏
  • 报告
  • 合作伙伴商店
  • 资源

开始使用

UDP 控制台的“开始使用”选项卡包含有关如何使用 UDP 控制台的有用信息，其中包括： * 教程视频 * 一个可帮助你跟踪进度的概要指南 * 特定于商店的信息 * 常见问题解答

我的游戏

“我的游戏”选项卡显示你的 UDP 项目。在这里，你可以在 UDP 项目之间切换、创建新游戏以及存档和恢复游戏。

选择一个游戏后，可看到以下选项卡：

  • 游戏信息
  • 发布
  • 状态

每个游戏卡都会显示游戏状态。“已发布”状态表示游戏已完成所有进行重新打包并提交到商店的可用步骤。

游戏信息

使用“游戏信息”页面可查看和编辑游戏信息。 要查看游戏的“游戏信息”页面，请在“我的游戏”选项卡中选择相应的卡。

“游戏信息”页面包含以下部分：

  • 游戏描述
  • 二进制文件
  • 广告
  • 付费价格
  • 应用内购
  • 沙盒测试
  • 应用签名
  • 集成信息

“游戏信息”页面包含一个进度条，显示离游戏发布目标还有多远。此外，为该部分添加了必要的信息时，侧边菜单中的选项卡将显示为勾选状态。

游戏描述

在“游戏描述”部分中可输入将在应用商店中显示的游戏信息。

下表介绍了所有目标商店的要求超集。并非所有商店都使用表中介绍的所有属性。UDP 控制台会标记哪些字段是必填字段，哪些是可选字段。此外，某些商店有特定的语法要求。这些信息在 UDP 控制台中有相应指示。

属性 描述
从 Google Play 导入 如果你已经在 Google Play 上发布了游戏，可使用游戏的 Google Play URL 导入基本信息。
游戏名称 游戏名称（该字段与编辑器同步）。
类型 表示游戏所属的类别。选项包括：动作游戏、冒险游戏、街机游戏、棋盘游戏、卡牌游戏、博彩游戏、休闲游戏、教育游戏、音乐游戏、解谜游戏、竞速游戏、角色扮演游戏、模拟经营游戏、体育游戏、策略游戏、益智问答游戏和文字游戏。
设备 选项包括：智能手机、平板电脑或通用。
游戏图标 要在应用商店中显示的游戏图标。
简短描述 要在应用商店中显示的游戏简短描述（最多 60 个字符）。
描述 要在应用商店中显示的游戏完整描述（4,000 个字符）。
游戏横幅 商店用来推荐游戏的图像。例如，可以是轮播中的一个展示位置。横向横幅是必选项，纵向横幅是可选项。
关键字 最多定义 4 个关键字（每个关键字最多 30 个字符）。这些关键字用于在应用商店中进行搜索。
介绍视频 为游戏添加视频预告片。
注意：对于视频，某些商店仅接受 MP4 文件，而其他商店仅接受 Youtube 链接。Unity 建议两种形式都上传。
屏幕截图 添加游戏的屏幕截图，包括封面图像（缩略图，用于在 MP4 视频无法播放时覆盖视频）。
每个使用方向至少添加四张图像。
一些应用商店仅接受纵向图像，或仅接受横向图像，而不接受同时两种方向的图像。如果将纵向图像和横向图像都上传到 UDP，请在偏好设置字段中选择首选方向。如果屏幕截图不符合商店的要求，UDP 将通知你。
偏好设置 某些商店仅接受纵向或横向图像。对于此类商店，请指定要使用的图像方向。
分级 选择游戏适合的受众。

可为每种支持的语言指定元数据。要添加新语言，请在编辑模式下选择语言下拉选单，然后选择“管理语言”。

二进制文件

在“二进制文件”部分中可上传 APK 文件和 OBB 文件：

属性 描述
APK 文件 你的 UDP 游戏构建版本。如果通过 Cloud Build 推送了 UDP 构建版本，则无需再次上传。
为使 UDP 接受 APK 文件，APK 文件：
- 必须包含 versionName
- 必须具有 Initialize() 方法
- 必须具有 Purchase() 方法（对于使用 IAP 的游戏）
如果上传的新 APK 版本具有不同的包名称，则会收到错误消息。还会阻止发布新的 APK 版本。
版本代码 仅当游戏已上传到 UDP 后，才会显示此字段。
最低 Android 版本支持 游戏支持的最低 Android 版本。仅当游戏已上传到 UDP 后，才会显示此字段。
新功能 描述此游戏版本的变化。UDP 会在商店中发布此信息。
仅当游戏已上传到 UDP 后，才会显示此字段。
OBB 文件（主文件） 游戏或应用程序可能需要的其他资源的主扩展文件。
OBB 文件（补丁） 这是一个可选文件，用于对主扩展文件进行小更新。
你的游戏是否使用 Google Play 服务？ 了解 Google Play 服务使用情况有助于 UDP 在提交阶段更好地为你提供指导。

在 CloudBuild 部署期间不会推送 OBB 文件，因此，如果游戏使用了 OBB 文件，则必须手动将它们上传到 UDP 控制台。

注意：只能在默认语言视图（英语）中更改 APK 文件和 OBB 文件。

广告

下表介绍了“广告”部分。

属性 描述
你的游戏是否包含广告？ 选择你的游戏是否包含广告。此信息可由 UDP 用于在提交过程中更好地为你提供指导。
你的游戏使用什么广告聚合解决方案？（可选） 如果游戏包含广告，请选择游戏使用哪种聚合解决方案。
你的游戏使用什么广告网络？（可选） 如果游戏包含广告，请选择游戏使用哪种广告网络。

某些商店希望你在提交给他们的游戏中采用他们专有的广告网络。

UDP 会询问你的游戏中是否包含广告，以便更好地了解你提交到商店的内容，尤其是 UDP 可以确定你的提交内容将因此被商店拒绝的情况下。

可能拒绝游戏的商店会在“发布”部分中显示警告图标，并提供工具提示以说明可能发生的问题。 否则，UDP 不会以任何方式修改游戏的广告实现。

对最常见的聚合层和广告网络的测试表明，为 UDP 商店重新打包的游戏通常在接收广告活动方面没有问题。请随时向 UDP 支持部门询问你的广告设置及其是否适合通过 UDP 分发。为了更快获得更可靠的答案，务必提及你的游戏使用哪种聚合 SDK 和广告网络。

付费价格

“付费价格”是玩家下载游戏时所需支付的价格。

属性 描述
管理金额和货币 设置游戏费用（美元）。
选择转换可自动将美元价格转换为其他列出的货币。你还可以手动编辑特定货币的价格。

注意：只能将付费游戏提交到支持付费游戏的商店。

应用内购

在“应用内购”部分中可添加和管理 IAP 商品。

如果使用 UDP 包，UDP 会自动使游戏的 IAP 目录在 Unity 编辑器和 UDP 控制台之间保持同步。

属性 描述
搜索框 按文本筛选 IAP 商品列表。
“类型”下拉选单 按类型（即消耗品或非消耗品）筛选 IAP 列表。
“排序”下拉选单 选择如何对 IAP 商品列表进行排序。
添加项目 单击此项可添加新的 IAP 商品。此时将打开一个窗口以输入 IAP 详细信息。
管理金额和货币 单击此项可设置其他货币的价格。
转换 将 IAP 商品的美元价格转换为全球货币。这将覆盖手动设置的任何其他货币价格。
常用货币 选择此项可打开“常用货币”窗口。要将货币设置为常用货币，请选择相应货币代码旁边的星形图标。这会将货币固定在列表的顶部。取消选择星形图标即可取消固定。
“IAP 项目”窗口

在“IAP 项目”窗口中可添加和编辑 IAP 商品的详细信息。要打开此窗口，请选择添加项目。 下表介绍了添加 IAP 项目窗口中的字段。

字段 描述
产品 ID 用于标识 IAP 商品的唯一 ID。
“产品 ID”必须满足以下要求：
- 以字母或数字开头
- 仅包含字母、数字、点 (.) 和下划线 (_)
- 不得使用大写字母
名称 IAP 商品的名称。
货币 显示价格所用的货币。
价格 IAP 商品的价格（美元 USD）。
必须指定一个价格，才能让玩家在游戏中购买商品。
消耗品 选择此选项表示 IAP 商品为消耗品。
描述 IAP 商品的简短描述。

沙盒测试

“沙盒测试”部分显示游戏的测试状态。在此部分中可创建用于在沙盒环境中进行测试的用户凭据，并会显示测试状态。 下表介绍了此部分的字段。

字段 描述
测试状态 显示游戏的测试状态。
UDP 初始化 显示初始化检查的测试状态。这项检查旨在确保调用 Initialize() 方法（适用于所有游戏）。
IAP 交易 显示 IAP 交易检查的测试状态。此情况下会调用 Purchase() 方法（仅适用于 IAP 游戏）
沙盒测试帐户 沙盒测试帐户的电子邮件地址。仅用作 UDP 沙盒环境的登录名。
密码 测试帐户的密码。

应用签名

UDP 使用应用签名私钥对提交到商店的重新打包的 APK 文件进行签名。

Unity 建议选择导出并上传密钥和证书，并使用你自己的应用签名私钥。如果在 Google Play 上发布了游戏，请使用与 Google Play 上相同的密钥。这样，当用户安装游戏时，游戏被 Google Play 保护机制标记的可能性将大大降低。

注意：如果游戏被标记，可在此处向 Google 提出申诉。

下表介绍了“游戏信息”选项卡的“应用签名”部分。

属性 描述
让 UDP 创建和管理应用签名私钥 UDP 为每个需要重新打包游戏的商店生成一个不同的应用签名私钥。
此选项会使你的游戏更容易受到 Google Play 保护警告的影响。
导出并上传密钥和证书（推荐） UDP 使用你上传的应用签名私钥对重新打包的构建版本进行签名。如果选择此选项，则会应用于你将游戏提交到的所有商店。
步骤 显示导出和上传私钥的步骤。

注意：只有 Android SDK 版本 18 及更高版本才支持生成应用签名。如果游戏使用的 SDK 版本低于此版本，请使用 RSA 算法签名或提高游戏的最低 API 级别 (Minimum API Level)。

更改密钥

你可以随时从使用 UDP 密钥切换到你自己的密钥。此更改仅适用于尚未重新打包游戏（因此未签名）的商店。如果先前已使用 UDP 密钥为某个商店重新打包了游戏，则该商店将始终使用 UDP 密钥。

仅当你还没有为任何商店重新打包游戏（使用你自己的密钥）时，才能从使用你自己的密钥切换为 UDP 密钥。

UDP 生成的密钥

满足以下条件时，UDP 使用公司信息为重新打包的 APK 文件生成密钥库：

  • 开发者上传的 APK 文件不包含密钥库文件。
  • 该游戏尚未在合作伙伴商店中重新打包。

如果不满足上述条件，则 UDP 会使用现有的 UDP 密钥库（如果可用），或者使用开发者信息来生成新的签名文件。

集成信息

此信息与编辑器同步。以下字段是可编辑的字段：

字段 描述
Unity 项目 ID 链接到 UDP 客户端的 Unity 项目 ID。
要取消 Unity 项目与 UDP 客户端的链接，请删除此字段中的 ID，然后选择保存。
注意：只有所有者和管理员可以编辑此字段。
回调 URL 游戏服务器的 URL。UDP 使用此 URL 发送有关付款结果的通知。

细节

如果提交中缺少任何内容，UDP 控制台会显示错误或警告消息。选择细节可展开有关该问题的更多信息。

错误显示在红色面板中。必须先修正错误，然后才能提交游戏。选择修改可进入错误区域以解决相关问题。

也可以选择只提交到没有错误的商店。

警告显示在黄色面板中。你可以关闭决定忽略的警告。

“状态”面板

UDP 开始处理你的游戏时，请访问状态面板来监控进度以及查看游戏的提交历史记录。

状态面板会显示游戏历史记录的概览。

字段 描述
已重新打包修订 使用 UDP 创建的重新打包的 APK 构建版本总数。
注意：如果针对同一家商店已将游戏重新打包三次，则计数为三次。
已提交修订 通过 UDP 提交的总次数。
注意：如果将游戏提交到商店五次，则计数为五次。
已接受提交 应用商店已接受的提交次数。
已拒绝提交 应用商店已拒绝的修订版数量。

对于每个游戏修订版，状态面板显示以下详细信息：

字段 描述
商店 将游戏提交到的商店。
状态 修订版状态。
并非所有商店都可以在进入“上线”状态之前提供可见性。“状态”部分仅提供可从商店获取的信息。
国家/地区 每个商店支持你游戏的国家/地区数量。选择此列中的数字可查看相应的国家/地区。
操作 执行其他操作，例如下载游戏的修订版或转到商店以完成发布步骤。

你的游戏可以具有以下状态：

  • 已重新打包：已成功使用选定商店的 SDK 将游戏重新打包
  • 已发布：游戏已完成了将游戏重新打包并提交到商店时该商店所需的所有步骤
  • 待处理：正在使用选定商店的 SDK 将游戏重新打包
  • 失败：无法使用选定商店的 SDK 将游戏重新打包
  • 已取消：你或你组织中的某个人取消了你提交的游戏

“报告”控制面板

“报告”选项卡显示已发布的游戏的表现信息。 你可以从“报告”控制面板监控已发布的游戏的性能（可从顶部导航栏访问该控制面板）。

你可以按游戏和/或按商店筛选“报告”控制面板上显示的信息。

如果没有在整个“报告”控制面板的所有游戏和所有商店筛选条件中选择任何选项，则可以查看单个图表级别的细分内容（如果适用）。在支持的图表上，选择按商店或按游戏筛选条件可查看特定商店或单个游戏的表现。

“报告”控制面板分为五个面板，所有面板均受“游戏”和“商店”筛选条件的约束。对于受日期筛选条件约束的面板，你可以将图表数据导出为 CSV 文件和图像。要将数据下载为 CSV 文件或图像，请选择图表的相应图标。

注意：仅当你使用 QueryOrder 设置了服务器端验证时，华为 AppGallery 才向 UDP 返回收入信息。如果你使用了回调通知，则华为没有与收入相关的图表或信息。

指标和定义

报告控制面板会跟踪发布游戏的 UDP 商店中的以下数据：

指标 描述
总收入 Amount collected from players in USD (e.g. if IAP price= 0.99 then= $0.99)
玩家 下载并启动游戏的唯一身份用户数。这是按照唯一设备计算的。
付费者 付费玩家的数量。也就是至少进行了一次成功付款的唯一身份用户数。
自第一次活动以来的天数 自游戏在应用商店中达到 10 位玩家以来的天数。
新玩家 在选定时间段内首次启动游戏的玩家数量。
DAU 日活跃用户。当天至少启动了一次游戏的唯一身份用户数。
MAU 月活跃用户。过去 30 天至少启动了一次游戏的唯一身份用户数。
IAP 收入 IAP 的总收入（美元）。
每笔交易的平均收入 定义为收入/交易数量（美元）。
ARPPU 付费用户的平均收入（美元）。定义为总收入/付费玩家数量。
ARPU 用户的平均收入（美元）。定义为总收入/玩家数量。
付费者转化率 定义为付费玩家数量/玩家数量。
交易 产生付款的成功交易数量（OrderID 成功并已由 UDP 服务器验证）。
IAP 收入 按 IAP 对总收入进行细分（美元）。
IAP 交易量 按 IAP 对交易进行细分。
游戏销售收入 付费游戏产生的总收入（美元）。
会话 在任何设备上启动游戏的次数。
第 1 天留存率 (%) 定义为首次启动游戏 1 天后仍活跃的玩家百分比。
第 7 天留存率 (%) 定义为首次启动游戏 7 天后仍活跃的玩家百分比。
第 30 天留存率 (%) 定义为首次启动游戏 30 天后仍活跃的玩家百分比。

注意：“报告”控制面板包含一些估计数据；你可以直接从商店获取确切的数字。显示的数据采用 UTC 时间，并且截止到前一天，旨在显示最准确的完整数据。

CSV 参考

CSV 文件的数据值

应用内购的 CSV 文件中的每一行都可以包含以下值。不要传递空值。

字段 描述
product_id 代表唯一 IAP 商品的 ID。product_id 需要以小写字母或数字开头，并且只能由小写字母 (a-z)、数字 (0–9)、下划线 (_) 和句点 (.) 组成。
language; product_name; description 设置 language 值时，请使用表 A 中列出的语言代码。
UDP 使用 en-US 作为默认语言。
使用分号 (;) 来分隔 language、product_name 和 description。
要包含商品的本地化商品名称和描述，必须列出默认 language、product_name 和 description，然后列出每种区域设置的 language、product_names 和 description。在以下示例中，商品的默认语言为 en-US (ENGLISH)，本地化语言为 zh-CN (CHINESE)：
en-US;Product 1;This is my first product; zh-CN; 产品1; 产品描述1。
注意：上传文件后，只会导入与“游戏信息”部分中已定义的语言相对应的字段。如果要支持多种语言，请在 UDP 控制台上定义这些语言，然后再导入 IAP 目录。
consumable 此值必须设置为 TRUE 或 FALSE（不区分大小写）。
currency; price 设置 currency 值时，请使用表 B 中列出的货币代码。price 必须为正。默认货币为美元 (USD)。每个 IAP 商品的价格必须直接列在相应的货币后面。例如：USD;0.99;CNY;6
