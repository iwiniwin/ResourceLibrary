升级资源包

资源包有两种升级：

  • Major：这种升级是现有资源包的新版本，并包含很大的功能差异。这可能表示有更多的资源，或者可能包括对 API 的重大更改。在这种情况下，原始资源包必须已经在 Asset Store 上存在一年以上。

    创建新的升级资源包后，您应该在 Asset Store 中弃用以前的版本，因为您希望用户找到新的资源包。已弃用的资源包仅可供以前购买或下载过这些资源包的客户使用。

  • Lite：这是针对较贵资源包提供的功能更少或资源更少的版本（通常是免费版）。通常，您需要创建一个“精简版”(lite) 资源包，让潜在客户试用您的资源包，以便他们决定是否要以全价购买另一个版本。

这些升级的过程大致相同：

1.创建新的资源包草案、向资源包上传资源以及将资源包提交到 Asset Store 进行发布。如果要创建 Major 升级，请确保对新资源包使用相同的名称。

2.打开 Asset Store Publisher 门户并导航到 Upgrades 选项卡。

![Paid Package Upgrades 选项卡](../uploads/Main/AssetStoreUpgrade.png)

3.单击 Create New Upgrade 按钮。此时会显示 Create Paid Package Upgrades 页面。

![Create Paid Package Upgrades 页面上的升级设置](../uploads/Main/AssetStoreUpgrade-create.png)

4.从 Type 下拉框中选择要创建的升级类型。

![选择升级类型](../uploads/Main/AssetStoreUpgrade-which.png)

5.从 Upgrade From 下拉菜单中选择原始资源包（至少一年前发布的资源包）。

6.从 Upgrade To 下拉菜单中选择新发布的资源包。

7.对于 Price USD 值，输入与您要创建的升级类型相对应的金额：

* 对于 Major 升级，请为购买了原始版本资源包的客户设置一个升级到新版本的价格。例如，与以前没有购买过该资源包的客户相比，已经购买了该资源包第 1 版的客户将为第 2 版支付更少的费用。
* 对于 Lite 升级，请设置一个更低的价格（或让您的资源包变为免费版），以体现精简版资源包与完整版相比的价值。

8.如果要创建 Major 升级，请输入 Grace Days 的天数，该天数表示新资源包发布之前的一段时间，客户在这段时间可以购买旧资源包并免费进行升级。例如，如果您设置 10 天宽限期 (Grace Days)，并且资源包的第 2 版在 3 月 20 日出现在了 Asset Store 中，那么在 3 月 10 日至 3 月 19 日之间购买第 1 版的任何用户都可以免费升级到第 2 版。

9.单击 Save，然后单击 Go Back 以返回到 Upgrades 选项卡。
